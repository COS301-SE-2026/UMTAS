    Branch Protection Setup Guide

    Written for the UMTAS 2026 Capstone Project

    Purpose

    To enforce a professional development workflow, we need to protect our main branch. This prevents accidental pushes, enforces code reviews, and ensures that all automated checks (CodeQL, Linting, Tests, Knip) pass before code is merged. This is a critical metric for capstone evaluators looking for mature engineering practices.

    Step-by-Step Setup

    1. Navigate to Repository Settings: Go to your GitHub repository page and click on Settings (the gear icon).

    2. Access Branches Section: In the left sidebar under "Code and automation", click on Branches.

    3. Add Branch Protection Rule: Click the Add branch protection rule button.

    4. Rule Configuration:

    5. Branch name pattern: Type main

    6. Protect Matching Branches: Check the following boxes:

    7. [x] Require a pull request before merging

      • [x] Require approvals (Set to at least 1)
      • [x] Dismiss stale pull request approvals when new commits are pushed
      • [x] Require review from Code Owners (if you decide to use a CODEOWNERS file later)

    8. [x] Require status checks to pass before merging

      • [x] Require branches to be up to date before merging
      • In the search bar for status checks, search for and add the following (they will only appear after they have run at least once on a PR):
      • build-and-test (from your CI pipeline)
      • Analyze (javascript-typescript) (from CodeQL)
      • Analyze (python) (from CodeQL)

    9. [x] Require conversation resolution before merging

      • This forces developers to acknowledge and resolve all comments on a PR before the merge button turns green.

    10. [x] Require signed commits (Optional but highly recommended for extra security points)

    11. Save Changes: Scroll to the bottom and click Create or Save changes.

    Verification

    To verify this works: 1. Try to push directly to main from your local terminal. It should be rejected. 2. Open a Pull Request. You should see that the merge button is blocked until a reviewer approves it and the GitHub Actions finish successfully.
