# DrizzleORM Reference Manual (v0.45.x)

## Section 0: Quick Start

Immediate path to a type-safe database connection.

```bash
# Install core ORM and CLI kit (Postgres example)
npm install drizzle-orm pg
npm install -D drizzle-kit @types/pg
```

```typescript
// db.ts - Minimal connection verification
import { drizzle } from "drizzle-orm/node-postgres";
import { pgTable, serial, text } from "drizzle-orm/pg-core";
const users = pgTable("users", {
  id: serial("id").primaryKey(),
  name: text("name"),
});
const db = drizzle(process.env.DATABASE_URL!);
console.log(await db.select().from(users));
```

Expected output: `[]` (empty array) or existing user rows from the database.

## Section 1: Key Language Terms & Features

- **TypeScript Schema** — Defining database tables and relations directly in TypeScript files | `export const users = pgTable('users', { id: serial('id').primaryKey() });` | ⚠️ Use classes/objects for schema organization to enable easy imports across the app.
- **Drizzle Kit** — The companion CLI tool for schema management and migrations | `drizzle-kit generate` | ⚠️ Always keep your `drizzle.config.ts` in the root for automatic discovery.
- **SQL Dialects** — Specialized cores for Postgres, MySQL, SQLite, and MSSQL | `import { pgTable } from 'drizzle-orm/pg-core';` | ⚠️ Don't mix dialect imports (e.g., using `mysqlCore` with a Postgres DB).
- **Migrations** — Version-controlled SQL files generated by diffing your TS schema against the DB | `drizzle-kit generate` | ⚠️ Review generated SQL files in the `drizzle/` folder before applying to production.
- **Schema Push** — Directly synchronizing schema with the DB without creating migration files | `drizzle-kit push` | ⚠️ Use for local development/prototyping; use migrations for production.
- **Relational Queries (RQB)** — High-level API for fetching nested data with optimized single SQL queries | `db.query.users.findMany({ with: { posts: true } });` | ⚠️ Ensure relations are defined in the schema using the `relations()` function.
- **Introspection** — Pulling an existing database schema into Drizzle TypeScript definitions | `drizzle-kit introspect` | ⚠️ Great for migrating legacy projects to Drizzle without manual typing.
- **Drizzle Studio** — A local web-based GUI for exploring and managing your data | `drizzle-kit studio` | ⚠️ Perfect for quick data entry or debugging without a heavy SQL client.
- **Strict Mode** — Configuration that prevents accidental data loss during schema changes | `strict: true` in config | ⚠️ Prevents "silent" column drops by requiring explicit user confirmation in the CLI.
- **Casing Mapping** — Automatically converting between `camelCase` (TS) and `snake_case` (SQL) | `casing: 'snake_case'` | ⚠️ Saves time by removing the need to manually specify every column name string.
- **Prepared Statements** — Pre-compiled queries for maximum performance and security | `const p = db.select().from(u).prepare();` | ⚠️ Use for frequently executed queries to reduce database parsing overhead.
- **Batch API** — Executing multiple queries in a single network round-trip | `await db.batch([q1, q2]);` | ⚠️ Crucial for reducing latency in serverless/edge environments like Cloudflare Workers.

## Section 2: Key Commands & Workflows

- `drizzle-kit generate` — Compares TS schema to migration files and creates new SQL scripts | _After modifying your schema files._
- `drizzle-kit push` — Updates the database schema directly to match your TS code | _Rapid prototyping in local dev environments._
- `drizzle-kit studio` — Launches the web-based database browser at localhost:4444 | _Viewing or editing data during development._
- `drizzle-kit introspect` — Connects to a DB and generates a `schema.ts` file | _When starting Drizzle on an existing database._
- `drizzle-kit check` — Validates that migration files match the current schema state | _CI/CD pipelines to prevent desynced migrations._
- `drizzle-kit up` — Updates older migration metadata to the current Drizzle version | _After upgrading drizzle-kit to a new major version._
- `drizzle-kit drop` — Removes a specific migration file from the project history | _When you need to undo an unapplied migration._

## Section 3: Architecture & Component Relationships

Drizzle acts as a thin, type-safe bridge between TypeScript and raw SQL.

```text
TypeScript Code (Schema)
      ↓
[ Drizzle Kit ] (Diffing & Generation)
      ↓
[ SQL Migrations ] (Version Control)
      ↓
[ Migration Tool / Push ] (Execution)
      ↓
[ Database (Postgres/MySQL/SQLite) ]
      ↑
[ Drizzle ORM (Runtime) ] (Querying & Mapping)
      ↑
[ TypeScript App (Business Logic) ]
```
