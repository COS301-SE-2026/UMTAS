# Design Brief

The Design Brief outlines the visual and structural design goals for the UMTAS project, providing a roadmap for the user interface and user experience.

---

## :material-target: Design Goals

1.  **Professional Clarity:** A clean, uncluttered interface suitable for academic and administrative use.
2.  **Information Density:** Optimizing screen real estate to show complex timetable data without overwhelming the user.
3.  **Visual Hierarchy:** Using typography and color to guide users through the scheduling and analytics workflows.

---

## :material-account-group: Target Audience

-   **Students:** Primary users focused on easy timetable creation and export.
-   **University Admins:** Power users needing deep analytics and dashboard control.
-   **Tyto Analysts:** Internal users performing high-volume simulations.

---

## :material-vector-combine: Structural Approach

-   **Dashboard-Centric:** Most interactions stem from a centralized dashboard.
-   **Modular Components:** Reusable UI elements for consistency across subsystems.
-   **Responsive Layout:** Adapting from desktop admin views to mobile student views.

---

## :material-clock-fast: Design Timeline

-   **Phase 1 (Demo 1):** Brand identity, basic wireframes, and layout foundations.
-   **Phase 2:** Interactive prototypes and user feedback integration.
-   **Phase 3:** Final visual polish and accessibility auditing.
